## 一、核心优化问题

### 1.1 目标函数

$$
\min_x \frac{1}{2}\|Ax - y\|_2^2 + \lambda_1 \|Wx\|_1 + \lambda_2 \|D \cdot \mathrm{Env}(x)\|_1
$$

其中：
- $A$：频域掩膜采样算子（`MaskedRFFT1D`）
- $W$：稀疏分析算子（固定 DWT 或可学习卷积，详见第四节）
- $D$：有限差分算子（固定 $[1, -1]$ 卷积核）
- $\mathrm{Env}(x) = |x + jH(x)|$：希尔伯特包络提取
- $\lambda_1, \lambda_2$：正则化参数（由 HASA 模块逐点自适应生成）

### 1.2 为什么 ISTA 不能处理这个问题

ISTA 的近端梯度下降要求计算联合近端映射：

$$
\mathrm{prox}_{\lambda_1\|W\cdot\|_1 + \lambda_2\|D \cdot \mathrm{Env}(\cdot)\|_1}(z)
$$

这个联合近端映射**没有闭式解**，原因：
1. 两个非光滑项作用在同一变量 $x$ 上，且不可分离
2. $\mathrm{Env}(x)$ 是非线性函数，使得第二项连凸性都失去了

### 1.3 ADMM 的解法：变量分裂

引入辅助变量 $w$ 和 $p$：

$$
\min_{x, w, p} \frac{1}{2}\|Ax - y\|_2^2 + \lambda_1 \|w\|_1 + \lambda_2 \|p\|_1
$$
$$
\text{s.t.} \quad w = Wx, \quad p = D \cdot \mathrm{Env}(x)
$$

**关键洞察**：L1 范数现在直接作用在辅助变量 $w$ 和 $p$ 上，每个子问题都有**精确闭式解（软阈值）**。

## 二、ADMM 展开网络的逐步推导

### 2.1 增广拉格朗日函数

$$
\mathcal{L} = \frac{1}{2}\|Ax - y\|_2^2 + \lambda_1\|w\|_1 + \lambda_2\|p\|_1 + \frac{\rho_1}{2}\|w - Wx + u_1\|_2^2 + \frac{\rho_2}{2}\|p - D\mathrm{Env}(x) + u_2\|_2^2
$$

其中 $u_1, u_2$ 是缩放的对偶变量，$\rho_1, \rho_2 > 0$ 是惩罚参数。

### 2.2 ADMM 迭代格式（第 $k$ 层）

**Step 1：w-update（稀疏近端）**

$$
w^{k+1} = \mathrm{soft}\!\left(W x^k + u_1^k,\; \frac{\lambda_1}{\rho_1}\right)
$$

- 精确的 $\|w\|_1$ 近端映射
- $W$ 可以是固定 DWT（Mode A）或可学习线性卷积（Mode B）
- **无需 $W$ 正交**——ADMM 通过变量分裂完全绕开了 ISTA 对正交性的要求

**Step 2：p-update（包络域 TV 近端）**

$$
p^{k+1} = \mathrm{soft}\!\left(D \cdot \mathrm{Env}(x^k) + u_2^k,\; \frac{\lambda_2}{\rho_2}\right)
$$

- 精确的 $\|p\|_1$ 近端映射
- $D$ 是固定的有限差分算子
- $\mathrm{Env}(\cdot)$ 是希尔伯特包络（FFT 实现，完全可微）
- **TV 平滑只作用在包络上**

**Step 3：x-update（数据保真 + 二次惩罚）**

$$
x^{k+1} = x^k - \eta^k \nabla_x \mathcal{L}\big|_{x=x^k}
$$

其中梯度为：
$$
\nabla_x \mathcal{L} = A^T(Ax - y) + \rho_1 W^T(Wx - w^{k+1} + u_1^k) + \rho_2 \cdot J_{\mathrm{Env}}^T(x) \cdot D^T\!\left(D\mathrm{Env}(x) - p^{k+1} + u_2^k\right)
$$

由于 $\mathrm{Env}(x)$ 的非线性，x-update 没有闭式解，用**一步梯度下降**（可学习步长 $\eta^k$）替代。

**Step 4：对偶变量更新**

$$
u_1^{k+1} = u_1^k + Wx^{k+1} - w^{k+1}
$$
$$
u_2^{k+1} = u_2^k + D\mathrm{Env}(x^{k+1}) - p^{k+1}
$$

### 2.3 深度展开：可学习参数一览

| 参数 | 经典 ADMM | 展开后 |
|------|-----------|--------|
| $\rho_1, \rho_2$ | 手动调参 | 逐层可学习标量 |
| $\eta$ (x-update 步长) | 无 | 逐层可学习标量 |
| $\lambda_1, \lambda_2$ | 全局常数 | **HASA 模块逐点自适应生成** |
| $W$ (稀疏分析算子) | 固定 DWT | **Mode A: 固定 / Mode B: 可学习** |
| $D$ (差分算子) | 固定 $[1,-1]$ | **保持固定**（保证物理可解释性） |

## 三、包络域 TV 的可微实现——精确算子，无需 CNN

### 3.1 核心结论

**在 ADMM 框架下，TV 的近端映射完全由精确的数学算子实现，不需要任何 CNN 来"模拟"。**

整个 TV 分支的数据流：

```
x (RF 信号)
  → hilbert_envelope(x)      # 希尔伯特包络提取 (FFT, 可微)
  → finite_diff(env)          # 有限差分 D·env (Conv1d with [1,-1], 可微)
  → soft_threshold(·, λ₂/ρ₂) # 软阈值 (逐元素, 可微)
  = p (TV 辅助变量)
```

每一步都是**精确的数学算子**。

### 3.2 各算子的可微实现

#### 3.2.1 希尔伯特包络提取

```python
def hilbert_envelope(x):
    """x: (B, 1, N) real RF → (B, 1, N) envelope"""
    x_flat = x.squeeze(1)                            # (B, N)
    X = torch.fft.fft(x_flat, dim=-1)                # 全谱 FFT
    N = X.shape[-1]

    # 构造解析信号的频域滤波器
    h = torch.zeros(N, device=x.device)
    h[0] = 1.0
    h[1:(N + 1) // 2] = 2.0
    if N % 2 == 0:
        h[N // 2] = 1.0

    analytic = torch.fft.ifft(X * h, dim=-1)         # 解析信号 (复数)
    envelope = torch.sqrt(
        analytic.real ** 2 + analytic.imag ** 2 + 1e-8
    )
    return envelope.unsqueeze(1)                      # (B, 1, N)
```

- FFT/IFFT：PyTorch 原生支持反向传播
- $\sqrt{a^2 + b^2 + \epsilon}$：处处可微（$\epsilon$ 避免零点奇异性）
- **不需要做除法**（`env_clean / env`），ADMM 框架下 TV 作用在 $p$ 变量上，不需要"调制回 RF"

#### 3.2.2 有限差分算子 $D$

```python
class FiniteDiff1D(nn.Module):
    """固定的一阶有限差分算子"""
    def __init__(self):
        super().__init__()
        kernel = torch.tensor([[[1.0, -1.0]]])  # (1, 1, 2)
        self.register_buffer('kernel', kernel)

    def forward(self, x):
        """x: (B, 1, N) → (B, 1, N-1)"""
        return F.conv1d(x, self.kernel)

    def adjoint(self, p):
        """D^T: (B, 1, N-1) → (B, 1, N)"""
        return F.conv_transpose1d(p, self.kernel)
```

#### 3.2.3 软阈值

```python
def soft_threshold(x, thr):
    return torch.sign(x) * F.relu(torch.abs(x) - thr)
```

### 3.3 为什么 ADMM 让 TV 实现变得简单

| 框架 | TV 子问题 | 求解方式 |
|------|----------|---------|
| **ISTA** | $\mathrm{prox}_{\lambda TV(\mathrm{Env}(\cdot))}(z)$：非线性非光滑 | 无闭式解，需迭代或 CNN 近似 |
| **ADMM** | $\mathrm{prox}_{\lambda\|\cdot\|_1}(D\mathrm{Env}(x) + u)$：逐元素 L1 | **软阈值，精确闭式解** |

ADMM 从根本上消除了对 CNN 模拟 TV 的需求。

## 四、稀疏分析算子 $W$——Mode A / Mode B 双模式设计

### 4.1 为什么 ADMM 不需要 $W$ 正交

**ISTA**：$\mathrm{prox}_{\lambda\|W\cdot\|_1}(z) = W^T \mathrm{soft}(Wz, \lambda)$ **仅当** $W^TW = I$。

**ADMM**：通过变量分裂 $w = Wx$，软阈值作用在 $w$ 空间：

$$
w^{k+1} = \mathrm{soft}(Wx^k + u^k, \lambda/\rho)
$$

这与 $W$ 的性质**完全无关**。$W$ 是正交的、过完备的、甚至非方阵的，w-update 都是精确的。

### 4.2 Mode A：固定 DWT（可解释性优先）

**用途**：Ablation Study，证明"即使用最简单的固定小波，ADMM 结构本身就带来增益"。

**实现**：固定 Haar 小波（可替换为 db4、sym4 等）。

```python
class HaarDWT1D(nn.Module):
    """1D Haar 小波分解（单层）"""
    def __init__(self):
        super().__init__()
        s2 = 1.0 / (2 ** 0.5)
        lo = torch.tensor([[[s2, s2]]])    # (1,1,2) 低通
        hi = torch.tensor([[[s2, -s2]]])   # (1,1,2) 高通
        self.register_buffer('lo', lo)
        self.register_buffer('hi', hi)

    def forward(self, x):
        """x: (B,1,N) → coeffs (B,1,N) [approx || detail]"""
        a = F.conv1d(x, self.lo, stride=2)   # (B,1,N//2)
        d = F.conv1d(x, self.hi, stride=2)   # (B,1,N//2)
        return torch.cat([a, d], dim=-1)      # (B,1,N)

    def inverse(self, w):
        """coeffs (B,1,N) → x (B,1,N)"""
        N_half = w.shape[-1] // 2
        a, d = w[..., :N_half], w[..., N_half:]
        return F.conv_transpose1d(a, self.lo, stride=2) + \
               F.conv_transpose1d(d, self.hi, stride=2)
```

**优势**：
- 参数可解释性最强——卷积核就是小波基
- 零额外参数
- 论文中可直接引用小波理论

**局限**：Haar 小波时频分辨率有限，可能不是超声信号的最优稀疏基。

### 4.3 Mode B：可学习稀疏分析算子（性能优先）

**用途**：追求最优重建质量，让网络学习最适合超声信号的稀疏表示。

**实现**：stride=2 的 `Conv1d`（分析）+ `ConvTranspose1d`（综合），均可学习。

```python
class LearnableAnalysis1D(nn.Module):
    """可学习的 1D 稀疏分析/综合算子对"""
    def __init__(self, num_filters=2, kernel_size=8):
        super().__init__()
        self.analysis = nn.Conv1d(
            1, num_filters, kernel_size,
            stride=2, padding=kernel_size // 2 - 1, bias=False
        )
        self.synthesis = nn.ConvTranspose1d(
            num_filters, 1, kernel_size,
            stride=2, padding=kernel_size // 2 - 1, bias=False
        )

    def forward(self, x):
        """x: (B,1,N) → coeffs (B, num_filters, N//2)"""
        return self.analysis(x)

    def inverse(self, w):
        """coeffs → x: (B,1,N)"""
        return self.synthesis(w)
```

**优势**：
- 学习超声信号的最优稀疏基，性能上限更高
- ADMM 框架保证 w-update 的精确性与 $W$ 的性质无关，理论仍自洽

**论文表述**：不称"学习小波变换"（因为没有 QMF/正交约束），而称"学习稀疏分析算子（Learned Sparse Analysis Operator）"。ADMM 允许 $W$ 是任意线性算子，这一点需在论文中明确说明。

### 4.4 Mode A/B 在网络中的切换

```python
class SparseTransform1D(nn.Module):
    """统一接口：固定 DWT 或可学习分析算子"""
    def __init__(self, mode='A', num_filters=2, kernel_size=8):
        super().__init__()
        self.mode = mode
        if mode == 'A':
            self.transform = HaarDWT1D()
        else:
            self.transform = LearnableAnalysis1D(num_filters, kernel_size)

    def forward(self, x):
        return self.transform(x)

    def inverse(self, w):
        return self.transform.inverse(w)
```

### 4.5 Ablation Study 设计

| 实验编号 | $W$ 配置 | 目的 |
|---------|---------|------|
| A1 | 固定 Haar | 基线：纯结构可解释性 |
| A2 | 固定 db4 | 验证更长小波基是否有增益 |
| B1 | 可学习, $K=4$, $C=2$ | 轻量可学习 |
| B2 | 可学习, $K=8$, $C=4$ | 更强表达能力 |

## 五、HASA 自适应权重嵌入 ADMM

### 5.1 HASA 的角色

HASA 生成逐点自适应的正则化参数 $\lambda_1(n)$ 和 $\lambda_2(n)$：

$$
w^{k+1}_n = \mathrm{soft}\!\left((Wx^k + u_1^k)_n,\; \frac{\lambda_1(n)}{\rho_1}\right)
$$
$$
p^{k+1}_n = \mathrm{soft}\!\left((D\mathrm{Env}(x^k) + u_2^k)_n,\; \frac{\lambda_2(n)}{\rho_2}\right)
$$

**数学合法性**：加权 L1 范数 $\sum_n \lambda_n |w_n|$ 的近端映射仍然是逐点软阈值。w/p-update 的精确性不受 $\lambda$ 的空间变化影响。

### 5.2 HASA 模块（ADMM 版）

ADMM 框架下不再需要输出 $\alpha$（混合权重），TV 和小波通过交替更新自然协作。

```python
class HASAWeight1D_ADMM(nn.Module):
    """ADMM 框架的 HASA 权重生成器"""
    def __init__(self, hidden_ch=16):
        super().__init__()
        self.feat_net = nn.Sequential(
            nn.Conv1d(1, hidden_ch, 3, padding=1),
            nn.ReLU(inplace=True),
            nn.Conv1d(hidden_ch, hidden_ch, 5, padding=2),
            nn.ReLU(inplace=True),
        )
        self.head_wav = nn.Sequential(
            nn.Conv1d(hidden_ch, 1, 1), nn.Softplus()
        )
        self.head_tv = nn.Sequential(
            nn.Conv1d(hidden_ch, 1, 1), nn.Softplus()
        )

    def forward(self, x):
        """x: (B,1,N) → lambda_wav (B,1,...), lambda_tv (B,1,...)"""
        feat = self.feat_net(x)
        return self.head_wav(feat), self.head_tv(feat)
```

注意：$\lambda_{\text{wav}}$ 的空间维度需与 $Wx$ 一致（Mode A 下为 $N$，Mode B 下为 $N/2 \times C$），$\lambda_{\text{tv}}$ 的空间维度需与 $D\mathrm{Env}(x)$ 一致（$N-1$）。HASA 输出在对应维度上做自适应池化或插值即可。

## 六、组合损失函数设计

### 6.1 设计原则

网络在 RF 域重建信号、在包络域施加 TV 约束。损失函数应同时保证两个域的质量：

$$
\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{RF}} + \gamma_{\text{env}} \cdot \mathcal{L}_{\text{Env}} + \gamma_{\text{sym}} \cdot \mathcal{L}_{\text{constraint}}
$$

### 6.2 各项损失

**① RF 域重建损失（物理保真）**

$$
\mathcal{L}_{\text{RF}} = \frac{1}{N}\|x_{\text{pred}} - x_{\text{gt}}\|_2^2
$$

保证 RF 信号的幅度、相位、载波频率的物理正确性。

**② 包络域感知损失（视觉质量）**

$$
\mathcal{L}_{\text{Env}} = \frac{1}{N}\|\mathrm{Env}(x_{\text{pred}}) - \mathrm{Env}(x_{\text{gt}})\|_1
$$

用 L1 而非 L2，理由：
- L1 对异常值更鲁棒
- 与 TV 的 L1 稀疏假设一致
- 更好地保持包络的锐利边缘

可选增强：替换为 Log L1 以匹配 B-mode 显示的对数压缩动态范围：
$$
\mathcal{L}_{\text{Env-Log}} = \frac{1}{N}\|\log(\mathrm{Env}(x_{\text{pred}}) + \epsilon) - \log(\mathrm{Env}(x_{\text{gt}}) + \epsilon)\|_1
$$

**③ ADMM 约束一致性损失（可选，辅助收敛）**

$$
\mathcal{L}_{\text{constraint}} = \|Wx^K - w^K\|_2^2 + \|D\mathrm{Env}(x^K) - p^K\|_2^2
$$

监督最后一层的辅助变量是否满足 ADMM 约束。如果网络层数足够，此项应趋近于零；如果层数有限，此项可辅助加速收敛。

### 6.3 超参数推荐

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| $\gamma_{\text{env}}$ | 0.1 ~ 0.5 | 包络域损失权重 |
| $\gamma_{\text{sym}}$ | 0.01 ~ 0.05 | 约束一致性权重（层数多时可设为 0） |
| $\epsilon$ (Log) | $10^{-6}$ | 对数稳定性 |

## 七、完整的单层 ADMM 展开结构（伪代码 v2）

```
HASA_ADMM_Block (第 k 层):

输入: x^k, w^k, p^k, u1^k, u2^k, y, op

  # ① HASA 自适应阈值
  λ_wav, λ_tv = HASA(x^k)

  # ② w-update: 稀疏近端 (精确, 可微)
  Wx = W.forward(x^k)                    # Mode A: DWT(x) / Mode B: Conv(x)
  w^{k+1} = soft_threshold(Wx + u1^k, λ_wav / ρ1^k)

  # ③ p-update: 包络域 TV 近端 (精确, 可微)
  env = hilbert_envelope(x^k)            # RF → 包络 (FFT, 可微)
  Denv = finite_diff(env)                # 包络的有限差分
  p^{k+1} = soft_threshold(Denv + u2^k, λ_tv / ρ2^k)

  # ④ x-update: 数据保真 + 二次惩罚 (梯度下降, 可学习步长)
  grad_data = op.At(op.A(x^k) - y)                   # ∇ 数据保真项
  grad_wav  = W.inverse(Wx - w^{k+1} + u1^k)         # ∇ 稀疏惩罚项
  grad_tv   = autograd(ρ2/2·||p^{k+1} - D·Env(x) - u2^k||², x)  # ∇ 包络TV惩罚
  x^{k+1}  = x^k - η^k · (grad_data + ρ1^k · grad_wav + grad_tv)

  # ⑤ 对偶更新
  u1^{k+1} = u1^k + W.forward(x^{k+1}) - w^{k+1}
  u2^{k+1} = u2^k + finite_diff(hilbert_envelope(x^{k+1})) - p^{k+1}

输出: x^{k+1}, w^{k+1}, p^{k+1}, u1^{k+1}, u2^{k+1}
```

### 逐层可学习参数

| 参数 | 含义 | 数量（每层） |
|------|------|-------------|
| $\eta^k$ | x-update 梯度步长 | 1 标量 |
| $\rho_1^k, \rho_2^k$ | ADMM 惩罚参数 | 2 标量 |
| HASA 网络权重 | 自适应阈值生成 | 每层独立或跨层共享 |
| $W$ 的权重（仅 Mode B） | 稀疏分析算子 | 每层独立或跨层共享 |

### 固定算子

| 算子 | 实现 | 可学习？ |
|------|------|:--------:|
| $A / A^T$ | `MaskedRFFT1D` | 否 |
| $W / W^T$（Mode A） | Haar DWT | 否 |
| $D / D^T$ | `[1, -1]` 卷积 | 否 |
| $\mathrm{Env}(\cdot)$ | FFT 希尔伯特包络 | 否 |
| $\mathrm{soft}(\cdot)$ | 逐元素操作 | 否 |

## 八、工程实现细节

### 8.1 x-update 中包络梯度的计算

**方案 A：PyTorch autograd（推荐，1D 信号下足够高效）**

```python
x = x_prev.clone().requires_grad_(True)
env = hilbert_envelope(x)
Denv = finite_diff(env)
residual_tv = Denv - p_new.detach() + u2.detach()
loss_tv = 0.5 * rho2 * (residual_tv ** 2).sum()
grad_tv = torch.autograd.grad(loss_tv, x, create_graph=True)[0]
```

**方案 B：手推解析梯度（备选，2D 扩展或显存紧张时使用）**

$\mathrm{Env}(x)$ 的雅可比可以解析推导。设解析信号 $a = x + jHx$，包络 $e = |a| = \sqrt{x^2 + (Hx)^2 + \epsilon}$，则：

$$
\frac{\partial e}{\partial x} = \frac{x + H^T(Hx)}{e} = \frac{x + H^T(Hx)}{\sqrt{x^2 + (Hx)^2 + \epsilon}}
$$

其中 $H^T = -H$（希尔伯特变换的伴随是其负值）。这个解析梯度可以避免 `create_graph=True` 的额外显存开销。

### 8.2 计算开销估算

每层 ADMM 的主要 FFT 操作：

| 步骤 | FFT 次数 | 来源 |
|------|:--------:|------|
| p-update: `hilbert_envelope` | 2 | fft + ifft |
| x-update: autograd 反向传播中的 `hilbert_envelope` | 2 | fft + ifft（反向） |
| 对偶更新: `hilbert_envelope` | 2 | fft + ifft |
| **每层合计** | **6** | |
| **K=9 层合计** | **54** | |

对于 1D 信号（$N \sim 2000\text{-}8000$），单次 FFT 在 GPU 上耗时 $< 0.1\text{ms}$。54 次 FFT $\approx 5\text{ms}$，相对于整个网络的前向传播时间可以忽略。

**显存方面**：`create_graph=True` 会保存 FFT 的中间计算图用于二阶梯度。对 1D 信号（batch × 1 × N），每层额外显存 $\approx 2 \times B \times N \times 8$ 字节（复数）。$B=16, N=4096$ 时约 1MB/层，9 层共 $\approx 9\text{MB}$，完全可接受。

**结论**：1D 场景下直接用 autograd，不需要手推解析梯度。如果未来扩展到 2D B-mode 图像，再考虑方案 B。

### 8.3 对偶更新的优化：减少重复计算

对偶更新中的 `W.forward(x^{k+1})` 和 `hilbert_envelope(x^{k+1})` 在下一层的 w-update 和 p-update 中会再次计算。可以缓存这两个结果避免重复计算：

```python
# 在对偶更新时计算并缓存
Wx_new = W.forward(x_new)
env_new = hilbert_envelope(x_new)
Denv_new = finite_diff(env_new)

u1_new = u1 + Wx_new - w_new
u2_new = u2 + Denv_new - p_new

# 传递给下一层，避免重复计算
# 下一层的 w-update 直接用 Wx_new，p-update 直接用 Denv_new
```

这样每层的 FFT 次数从 6 降为 4（省去对偶更新中的重复 `hilbert_envelope`）。

### 8.4 信号长度对齐

| 算子 | 输入维度 | 输出维度 | 注意事项 |
|------|---------|---------|---------|
| `hilbert_envelope` | $(B,1,N)$ | $(B,1,N)$ | 长度不变 |
| `finite_diff` | $(B,1,N)$ | $(B,1,N-1)$ | 缩短 1 |
| Haar DWT (Mode A) | $(B,1,N)$ | $(B,1,N)$ | concat(approx, detail) |
| Learnable Conv (Mode B) | $(B,1,N)$ | $(B,C,N/2)$ | 下采样，$C$ 为滤波器数 |

对偶变量维度：
- $u_1$：与 $w$ 一致（Mode A: $(B,1,N)$, Mode B: $(B,C,N/2)$）
- $u_2$：与 $p$ 一致（$(B,1,N-1)$）

### 8.5 辅助变量的初始化

```python
x0 = op.At(y)                                   # 零填充逆变换
w0 = W.forward(x0)                               # 初始稀疏系数
p0 = finite_diff(hilbert_envelope(x0))           # 初始包络差分
u1_0 = torch.zeros_like(w0)                      # 对偶变量初始化为零
u2_0 = torch.zeros_like(p0)
```

## 九、新架构 vs 旧架构的完整对比

| 维度 | 旧架构 (HASA-ISTA) | 新架构 (HASA-ADMM) |
|------|--------------------|--------------------|
| **优化框架** | ISTA / FISTA | ADMM |
| **双正则处理** | 并行分支 + α混合 | 变量分裂 + 交替更新 |
| **数学合法性** | $\mathrm{Prox}_{g+h} \neq \alpha\mathrm{Prox}_g + (1-\alpha)\mathrm{Prox}_h$ ❌ | 每个子问题精确闭式解 ✅ |
| **TV 实现** | CNN (E/D) 黑盒近似 | 精确算子: $D$ + soft\_threshold |
| **TV 作用域** | RF 信号域 ❌ | 包络域 ✅ |
| **稀疏分析 $W$** | 学习型 CNN (E/D) | Mode A: 固定 DWT / Mode B: 可学习线性卷积 |
| **正交性要求** | 需要但未满足 ❌ | 不需要 ✅ |
| **ReLU 非线性** | 在分支中, 破坏理论 | 不出现在 w/p-update 中 |
| **HASA 输出** | λ\_tv, λ\_wav, α (三项) | λ\_tv, λ\_wav (两项) |
| **损失函数** | 仅 RF MSE | RF MSE + 包络域 L1 + 约束一致性 |
| **可解释性** | 结构 + 参数均存疑 | 结构清晰 + 参数可解释（Mode A）或结构清晰（Mode B） |
| **参数量** | TV 分支 4 层 Conv + WAV 分支 4 层 Conv | HASA + 少量标量（Mode A）/ HASA + 稀疏算子（Mode B） |

## 十、论文叙事建议

### 10.1 命名

- 方法名：**HASA-ADMM-Net**
- 全称：Hybrid Adaptive Sparse Analysis ADMM Network for Ultrasound RF Signal Reconstruction

### 10.2 可解释性的表述

> "本文提出的 HASA-ADMM-Net 的每一层都严格对应 ADMM 算法的一次完整迭代。其中：
> - w-update 执行明确的小波域（或学习稀疏域）软阈值去噪；
> - p-update 在希尔伯特包络域执行全变分平滑；
> - x-update 融合数据保真与先验约束。
>
> 网络学习的仅仅是正则化强度的空间分布（由 HASA 模块生成的逐点阈值），而非去噪操作本身的形式——后者由经典的软阈值算子精确给出。"

### 10.3 回应审稿人的要点

| 审稿意见 | 回应 |
|---------|------|
| 双分支并行 ≠ 联合近端 | ADMM 变量分裂，w/p-update 均为精确闭式解 |
| TV 不应作用于 RF 域 | TV 明确作用于希尔伯特包络 |
| CNN 模拟 TV 是黑盒 | 用精确的 $D$ + soft\_threshold，无 CNN |
| 卷积核缺乏正交性 | ADMM 不需要 $W$ 正交；Mode A 使用固定 DWT 保证可解释性 |
| 动态 λ 破坏收敛 | w/p-update 精确性不受 λ 空间变化影响 |
| 可解释性边界 | 结构可解释（ADMM 步骤）+ 参数可解释（固定算子，Mode A） |
| 损失函数与物理不一致 | RF MSE 保证物理正确，包络 L1 保证视觉质量 |
| 表达能力可能退化 | Mode B 提供可学习稀疏算子，ablation study 对比两种模式 |
